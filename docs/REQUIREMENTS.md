# Требования

## Сущности

| Сущность | Описание |
|----------|----------|
| Группа | Учебная группа / поток |
| Дисциплина | Предмет с количеством часов в неделю |
| Преподаватель | Исполнитель занятий, доступность по слотам |
| Аудитория | Помещение, вместимость, тип (лекция, лаборатория) |
| Временной слот | День недели + пара (или интервал) |

## Ограничения

### Жёсткие

- Один преподаватель не может вести два занятия в одном слоте
- Одна группа не может быть на двух занятиях одновременно
- Аудитория не занята двумя группами в одном слоте
- Занятие назначается только в допустимые слоты (доступность преподавателя, тип аудитории)

### Мягкие (оптимизация)

- Минимизация «окон» у групп и преподавателей
- Равномерное распределение нагрузки по дням
- Предпочтительные аудитории и время

## Вход / выход

- **Вход:** структурированный файл (формат уточняется: JSON / CSV / Excel)
- **Выход:** расписание по группам и/или по преподавателям, отчёт о нарушениях и неразмещённых занятиях
